# Сценарий демо (2–5 минут)

Перед записью или живым показом: **меню профиля (шапка) → «Сбросить данные демо»** или кнопка на экране входа — так чистятся только данные в браузере (задачи из формы, отклики, смены статусов). Зашитые примеры в коде останутся.

## Маршрут

1. **Landing** — кратко: безопасная подработка, две роли, кнопка «Открыть демо».
2. **Вход** — `demo_employer` / `demo123` (или карточка «Работодатель»).
3. **Главная работодателя** — «Создать задачу».
4. **Новая задача** — заполнить форму (уже с подсказками), **Опубликовать**.
5. **Мои задачи** — убедиться, что задача в списке; открыть карточку.
6. **Выйти** → снова **Вход** — `demo_teen` / `demo123`.
7. **Задачи** (каталог) — найти только что созданную задачу (поиск/скролл).
8. **Карточка задачи** — **Откликнуться** (тост + блок успеха).
9. **Отклики** — новый отклик сверху (сортировка по дате).
10. **Профиль** — обновились цифры / активность (с учётом демо-данных в коде).
11. Снова **Вход** как **работодатель** → открыть ту же задачу.
12. **Отклики по задаче** — **Принять в работу** → (как подросток) **Отклики** → **Отметить выполненным** → снова работодатель → **Подтвердить оплату**.
13. **Подросток** — **Отклики** статус «Оплачено», **Профиль** / **Главная** — кошелёк и история.
14. Финальная фраза: «Сценарий замкнут: от публикации до выплаты в демо».

## Для скринкаста

- Держать один браузер/профиль; не смешивать с реальным кэшем без сброса.
- При ошибке — сброс и повтор с шага 2.
- Длинные паузы не нужны: тосты и карточки успеха подсказывают результат.

## User testing

Тот же маршрут; наблюдатель фиксирует, где участник ищет «что дальше». Точки риска: переключение роли (вход), поиск новой задачи в каталоге, цепочка статусов на стороне работодателя.
